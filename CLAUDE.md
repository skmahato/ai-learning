# AI Learning - Project Context

## About the Learner
- **Background:** 8 years experience as a frontend developer (React, JavaScript), some backend Node.js
- **Goal:** Transition into an AI Engineer role
- **Strategy:** Leverage existing full-stack skills and add AI as a new capability layer — NOT pivoting into ML research or data science

## What's Here
- `ai-engineer-roadmap.html` — Interactive syllabus/roadmap with 9 phases (0-8), covering Foundations through Specialization. Has checkboxes for progress tracking and notes per topic, all persisted in localStorage. Open in a browser to use.

## Roadmap Phases (Quick Reference)
0. **Foundations** (1-2 weeks) — Python basics, stats refresher
1. **LLM Fundamentals** (2-3 weeks) — How transformers work, model landscape
2. **Prompt Engineering** (2-3 weeks) — Design patterns, reliability, advanced techniques
3. **AI-Powered Applications** (3-4 weeks) — Full-stack AI apps with React + Node.js
4. **RAG** (3-4 weeks) — Embeddings, vector DBs, document processing, pipeline design
5. **Tool Use & Function Calling** (2-3 weeks) — Making LLMs take actions
6. **AI Agents** (3-4 weeks) — ReAct, frameworks, MCP, multi-agent systems
7. **Production & Operations** (2-3 weeks) — Eval, observability, cost, reliability
8. **Specialization** (Ongoing) — Multimodal, fine-tuning, voice AI, domain-specific

Total timeline: ~5-6 months

## How to Help
- When the user asks about a topic, tailor explanations for someone with deep React/JS/Node experience
- Use JavaScript/TypeScript examples over Python when possible, unless the topic requires Python
- Each phase has a hands-on project — help build these when asked
- The user prefers practical, build-it-yourself learning over theory-heavy study
